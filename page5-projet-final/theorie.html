Architecture MVC

Le projet utilise l’architecture MVC pour séparer la logique de gestion des données (models), les routes (controllers) et les endpoints (routes).

Bases de données

  • MySQL direct avec `mysql2/promise` pour requêtes SQL classiques
  • Sequelize pour ORM et simplification des relations
  • MongoDB avec Mongoose pour un stockage NoSQL flexible

Endpoints REST

Chaque ressource (User, Project) aura ses endpoints : GET, POST, PUT, DELETE. Les entrées seront validées et les erreurs correctement gérées.

Validation & sécurité

Les données envoyées par le client doivent être vérifiées. Les codes HTTP corrects seront renvoyés pour signaler succès, erreur client ou erreur serveur.

Test & Postman

Chaque route sera testée avec Postman pour s’assurer de la conformité et de la cohérence des résultats.
